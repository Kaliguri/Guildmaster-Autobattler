доп инфа про структурность ai 

Filter / Score / Override

## Filter

```
Цель вообще допустима?
```

Примеры:

```
Только враги  
Только союзники  
Только живые  
Только в радиусе X  
Только без эффекта X  
Только с эффектом X  
Только из списка целей  
Только HP < X
```

## Score

```
Насколько цель хороша?
```

Примеры:

```
Чем ближе, тем лучше
Чем ниже HP, тем лучше
Чем выше max HP, тем лучше
Чем ниже маг броня, тем лучше
Чем ниже физ броня, тем лучше
Если цель из списка — +Score
Если цель без эффекта X — +Score
Если цель является боссом — +Score
```
--- 

## Override

```
Что перебивает обычный выбор?
```

Примеры:

```
Если self HP < 10% → отступать  
Если союзник HP < 15% → хилить  
Если враг HP < 5% и в радиусе → добить  
Если опасный враг ближе X → отойти  
Если способность может задеть 4+ цели → кастовать
```

Судьба должна не ограничивать геймплей, а по своему раскрывать его. Например.

Судьба "Элита". Самая обычная судьба, дающая небольшой бонус к статам. Все

Судьба "Ученый". В начале имеет дебаффы с своим статам. Но за каждую уникальную реликвию, с которой он дрался - он получает бонус "Компетентности". При битве с реликвией, которую он уже брал - он получает бонус к статам за каждый уровень компетентности (но это не дает ему роста силы как такового).

Судьба "Нестабильный". Получает большую скорость перезарядки умений, но умирает через Х секунд боя. При смерти - наносит ВСЕМ вокруге Y урона.
